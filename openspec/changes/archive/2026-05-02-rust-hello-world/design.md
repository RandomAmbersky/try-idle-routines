## Context

Репозиторий `try-idle-routines` задуман как idle RTS на Rust; в корне пока может не быть `Cargo.toml` (`openspec/project.md`). Это изменение вводит минимальный бинарный крейт как основу для дальнейшей разработки.

## Goals / Non-Goals

**Goals:**

- Один исполняемый бинарь в корне (`cargo run` печатает предсказуемую строку в stdout).
- Стандартная структура Cargo: `Cargo.toml` с именем пакета, согласованным с репозиторием, и `src/main.rs` как единственная точка входа на этом этапе.
- Сборка и запуск без внешних зависимостей (только `std`).

**Non-Goals:**

- Игровая логика, симуляция, рендер, WASM, плагины.
- Настройка CI, MSRV, `rustfmt`/`clippy` в конфигах — только упоминание в задачах при необходимости.
- Библиотечный крейт, workspace из нескольких крейтов, бенчмарки.

## Decisions

| Решение | Обоснование | Альтернативы |
|--------|-------------|--------------|
| Один **binary** крейт в корне | Проще всего для «hello world» и соответствует типичному `cargo new --bin` | Workspace с `crates/game` — избыточно сейчас |
| Имя пакета **`try-idle-routines`** (или близкое к имени репо) | Единообразие с `openspec/project.md` | Короткое имя вроде `idle-rts` — возможна путаница с папкой репо |
| Текст вывода: **`Hello, world!`** (латиница, запятая, восклицательный знак) | Де-факто стандарт для проверки и тестов | Локализованная строка — усложняет сравнение в тестах |

## Risks / Trade-offs

- **[Risk]** Случайно зафиксировать слишком жёсткую строку в спеке → **Mitigation:** в спеке зафиксировать минимум «одна строка приветствия в stdout» или точную строку; в задачах — `cargo run` как ручная проверка.
- **[Risk]** `Cargo.lock` для бинаря: коммитить или нет → **Mitigation:** для приложения в одном крейте принято коммитить `Cargo.lock`; добавить при первом `cargo build`.

## Migration Plan

Не применимо: нового развёртывания нет. После merge разработчики выполняют `cargo build` / `cargo run` локально.
